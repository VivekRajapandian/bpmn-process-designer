# App Component Files Explained

Files:

- `src/app/app.component.ts`
- `src/app/app.component.html`
- `src/app/app.component.scss`
- `src/app/app.component.spec.ts`

These are the root Angular component files.

Spring Boot comparison:

The root Angular component is like the top-level entry view of the frontend app. It is not a backend controller. It is the first UI component Angular renders inside the browser.

## `app.component.ts`

```ts
import { Component } from '@angular/core';
import { BpmnWorkspaceComponent } from './bpmn-workspace/bpmn-workspace.component';

@Component({
  selector: 'app-root',
  imports: [BpmnWorkspaceComponent],
  templateUrl: './app.component.html',
  styleUrl: './app.component.scss'
})
export class AppComponent {
  title = 'BPMN Process Designer';
}
```

### Imports

```ts
import { Component } from '@angular/core';
```

This imports Angular's `Component` decorator.

```ts
import { BpmnWorkspaceComponent } from './bpmn-workspace/bpmn-workspace.component';
```

This imports the main BPMN workspace screen.

## Component Metadata

```ts
@Component({
  selector: 'app-root',
  imports: [BpmnWorkspaceComponent],
  templateUrl: './app.component.html',
  styleUrl: './app.component.scss'
})
```

### `selector: 'app-root'`

This defines the root HTML tag for the Angular app.

Angular starts by rendering:

```html
<app-root></app-root>
```

That root element is normally located in `src/index.html`.

Spring Boot comparison:

This is like the main mounted entry point for the frontend application. In Spring Boot terms, it is closer to the first page shell than a REST endpoint.

### `imports: [BpmnWorkspaceComponent]`

The root component template uses the BPMN workspace component.

Because this project uses standalone Angular components, the root component imports `BpmnWorkspaceComponent` directly.

### `templateUrl`

```ts
templateUrl: './app.component.html'
```

This points to the HTML file for the root component.

### `styleUrl`

```ts
styleUrl: './app.component.scss'
```

This points to the SCSS file for root-level styling.

## Class

```ts
export class AppComponent {
  title = 'BPMN Process Designer';
}
```

The class is very small because the real app behavior is inside `BpmnWorkspaceComponent`.

`title` is a basic property generated by Angular. It can be used by tests or templates, but the POC's main UI is the workspace.

## `app.component.html`

```html
<app-bpmn-workspace />
```

This is the whole root template.

It says:

"Render the BPMN workspace as the main application screen."

This works because `BpmnWorkspaceComponent` is listed in the `imports` array of `app.component.ts`.

## `app.component.scss`

This file is available for root component styling.

Most actual layout styling is inside the feature component SCSS files, especially:

- `bpmn-workspace.component.scss`
- `toolbar.component.scss`
- `bpmn-canvas.component.scss`

## `app.component.spec.ts`

This is the Angular unit test file for the root component.

It is generated by Angular CLI. For this POC, the main milestone focus is running/building the app and validating BPMN integration. Future milestones can add deeper component and service tests.

## Flow

```text
Angular bootstraps app
  -> creates AppComponent
  -> app.component.html renders <app-bpmn-workspace>
  -> BpmnWorkspaceComponent creates the actual BPMN modeler page
```

## Why This File Matters

The root app component keeps the application shell clean.

It delegates real BPMN modeler behavior to the workspace instead of putting everything in `AppComponent`.
